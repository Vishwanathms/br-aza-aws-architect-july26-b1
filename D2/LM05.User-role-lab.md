Here’s a hands-on AWS IAM lab for **Role-Based Access Control (RBAC)** within the same AWS account.

We’ll create:

* **IAM User** → `lab-user`
* **IAM Role 1** → `DeveloperRole` (S3-only access)
* **IAM Role 2** → `TesterRole` (EC2-only access)

Concept:
The user logs in and **switches roles** based on the task.

Use Amazon Web Services IAM.

---

# Lab Objective

| Entity        | Access                |
| ------------- | --------------------- |
| lab-user      | No direct permissions |
| DeveloperRole | Access only to S3     |
| TesterRole    | Access only to EC2    |

---

# Architecture

```text
IAM User (lab-user)
       |
       | AssumeRole
       |
   ----------------------
   |                    |
DeveloperRole       TesterRole
(S3 Only)           (EC2 Only)
```

---

# Step 1: Create IAM User

Go to:

Console → IAM → Users → Create User

Name:

```text
lab-user
```

Options:

* Enable AWS Management Console access
* Set custom password

Do NOT attach permissions.

Result:

```text
lab-user = No permissions
```

---

# Step 2: Create Developer Role

Go to:

IAM → Roles → Create Role

Choose:

```text
AWS Account
```

Account ID:

Use your same account ID.

Enable:

```text
Require MFA = No
```

Role name:

```text
DeveloperRole
```

---

## Attach S3 Policy

Use AWS managed policy:

```text
AmazonS3FullAccess
```

(For lab simplicity)

Production better:

Custom bucket-specific policy.

---

# Step 3: Edit Trust Policy for DeveloperRole

Open role → Trust relationships → Edit.

Replace with:

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": {
        "AWS": "arn:aws:iam::<ACCOUNT-ID>:user/lab-user"
      },
      "Action": "sts:AssumeRole"
    }
  ]
}
```

Replace:

```text
<ACCOUNT-ID>
```

with your account.

This means:

Only `lab-user` can switch into DeveloperRole.

---

# Step 4: Create Tester Role

IAM → Roles → Create Role

Choose:

```text
AWS Account
```

Same account.

Role name:

```text
TesterRole
```

---

## Attach EC2 Policy

Use:

```text
AmazonEC2FullAccess
```

---

# Step 5: Edit Trust Policy for TesterRole

Use:

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": {
        "AWS": "arn:aws:iam::<ACCOUNT-ID>:user/lab-user"
      },
      "Action": "sts:AssumeRole"
    }
  ]
}
```

Same idea.

---

# Step 6: Give User Permission to Switch Roles

Now attach inline policy to `lab-user`

IAM → Users → lab-user → Add permissions → Create inline policy

Policy:

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": "sts:AssumeRole",
      "Resource": [
        "arn:aws:iam::<ACCOUNT-ID>:role/DeveloperRole",
        "arn:aws:iam::<ACCOUNT-ID>:role/TesterRole"
      ]
    }
  ]
}
```

This allows switching.

# Validation Commands (CLI)

Check current identity:

```bash
aws sts get-caller-identity
```

Assume developer role:

```bash
aws sts assume-role \
--role-arn arn:aws:iam::<ACCOUNT-ID>:role/DeveloperRole \
--role-session-name devsession
```

Assume tester role:

```bash
aws sts assume-role \
--role-arn arn:aws:iam::<ACCOUNT-ID>:role/TesterRole \
--role-session-name testsession
```
