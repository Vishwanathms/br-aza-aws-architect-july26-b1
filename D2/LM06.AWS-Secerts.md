Here’s a practical AWS Secrets Manager lab with IAM role attachment to EC2.

Using Amazon Web Services and AWS CLI.

---

# Lab Scenario: Secure Database Password Retrieval from EC2

## Scenario

Your application runs on an EC2 instance and needs a DB password.

Instead of hardcoding:

❌ Bad:

```text
DB_PASSWORD=MySuperSecret123
```

Use:

✅ AWS Secrets Manager

Flow:

```text
EC2 Instance
   |
IAM Role attached
   |
Read Secret
   |
AWS Secrets Manager
```

Goal:

From inside EC2:

```bash
aws secretsmanager get-secret-value ...
```

Retrieve secret successfully.

---

# Lab Architecture

```text
Secret: prod/db/password
        |
        |
IAM Policy (Read Secret)
        |
IAM Role (EC2SecretsRole)
        |
Attached to
        |
EC2 Instance
```

# Lab Objective

You will:

* Create a secret
* Create IAM role
* Create EC2 instance
* Attach role
* Retrieve secret from EC2

---

# Step 1: Create Secret

Go to:

AWS Console → Secrets Manager → Store a new secret

Choose:

```text
Other type of secret
```

Add key/value:

| Key      | Value           |
| -------- | --------------- |
| username | admin           |
| password | MySecretPass123 |

Secret name:

```text
prod/db/password
```

Create.

---

# Step 2: Create IAM Policy for Secret Access

Go to:

IAM → Policies → Create policy

Use:

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "secretsmanager:GetSecretValue"
      ],
      "Resource": "arn:aws:secretsmanager:<REGION>:<ACCOUNT-ID>:secret:prod/db/password*"
    }
  ]
}
```

Name:

```text
SecretsReadPolicy
```

This gives read-only access.

---

# Step 3: Create IAM Role for EC2

Go to:

IAM → Roles → Create role

Trusted entity:

```text
AWS Service
```

Use case:

```text
EC2
```

Attach:

```text
SecretsReadPolicy
```

Role name:

```text
EC2SecretsRole
```

Create.

---

# Step 4: Launch EC2 Instance

Go to:

EC2 → Launch instance

Name:

```text
secret-lab-server
```

Recommended:

* Amazon Linux 2023
* t2.micro
* Allow SSH (22)

Under IAM Instance Profile:

Select:

```text
EC2SecretsRole
```

Launch.

---

# Step 5: Connect to EC2

SSH:

```bash
ssh -i mykey.pem ec2-user@<public-ip>
```

Verify AWS identity:

```bash
aws sts get-caller-identity
```

Expected:

Shows assumed role:

```text
EC2SecretsRole
```

---

# Step 6: Read Secret Using CLI

Command:

```bash
aws secretsmanager get-secret-value \
--secret-id prod/db/password \
--region <REGION>
```

Expected output:

```json
{
  "ARN": "...",
  "Name": "prod/db/password",
  "SecretString": "{\"username\":\"admin\",\"password\":\"MySecretPass123\"}"
}
```

Success.

---

# Step 7: Extract Only Password

Install jq:

```bash
sudo yum install jq -y
```

Get only secret string:

```bash
aws secretsmanager get-secret-value \
--secret-id prod/db/password \
--query SecretString \
--output text
```

Parse password:

```bash
aws secretsmanager get-secret-value \
--secret-id prod/db/password \
--query SecretString \
--output text | jq -r '.password'
```

Expected:

```text
MySecretPass123
```

# Validation Test

Try listing secrets:

```bash
aws secretsmanager list-secrets
```

Expected:

❌ Access denied (if policy only allows GetSecretValue)

Good security check.

---

# Failure Scenario Testing

Detach role from EC2.

Run:

```bash
aws secretsmanager get-secret-value --secret-id prod/db/password
```

Expected:

```text
Unable to locate credentials
```

This proves role-based auth.
